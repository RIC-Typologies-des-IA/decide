  <name>Task 1: Initialize Next.js project with Prisma</name>
  <files>package.json, prisma/schema.prisma, .env.example, .env.local</files>
  <action>
Initialize a new Next.js 14+ project with TypeScript and App Router:
1. Run `npx create-next-app@latest decide --typescript --tailwind --eslint --app --src-dir --import-alias "@/*"`
2. Initialize Prisma: `npx prisma init`
3. Install dependencies: `npm install @prisma/client bcryptjs jsonwebtoken resend zod`
4. Install dev dependencies: `npm install -D prisma @types/bcryptjs @types/jsonwebtoken`
5. Create `.env.example` with template variables:
   - DATABASE_URL
   - JWT_SECRET
   - RESEND_API_KEY
   - NEXT_PUBLIC_APP_URL
6. Copy to `.env.local` with development values

Avoid: Using pages router (App Router is the future), JS instead of TypeScript (type safety critical for auth)
  </action>
  <verify>
    <automated>npm list next prisma @prisma/client && cat package.json | grep -E '"next"|"prisma"'</automated>
  </verify>
  <done>package.json contains Next.js, Prisma, and auth dependencies; .env.example and .env.local exist with required variables</done>
</task>

<task type="auto">
  <name>Task 2: Create Docker Compose for PostgreSQL</name>
  <files>docker-compose.yml</files>
  <action>
Create docker-compose.yml for local PostgreSQL:
- Service name: postgres
- Port: 5432:5432
- Environment: POSTGRES_USER, POSTGRES_PASSWORD, POSTGRES_DB=decide
- Volume: postgres_data for persistence
- Healthcheck to ensure DB is ready

Update .env.local DATABASE_URL to point to local Docker PostgreSQL:
`postgresql://postgres:password@localhost:5432/decide?schema=public`

Start the container: `docker-compose up -d`
  </action>
  <verify>
    <automated>docker-compose ps | grep postgres && docker-compose exec -T postgres pg_isready -U postgres</automated>
  </verify>
  <done>PostgreSQL container is running and accepting connections on port 5432</done>
</task>

  <action>
Define the User model in prisma/schema.prisma with fields:
- id: String @id @default(uuid())
- email: String @unique
- passwordHash: String
- name: String?
- emailVerified: DateTime?
- resetToken: String? @unique
- resetTokenExpiry: DateTime?
- createdAt: DateTime @default(now())
- updatedAt: DateTime @updatedAt

Add indexes on email (unique already implies index) and resetToken.

Run initial migration:
`npx prisma migrate dev --name init_user_model`

Generate Prisma client:
`npx prisma generate`

Create src/lib/prisma.ts singleton for database client.
