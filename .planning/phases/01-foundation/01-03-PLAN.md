<task type="auto">
  <name>Task 2: Create login endpoint with cookies</name>
  <files>src/app/api/auth/login/route.ts</files>
  <action>
Create src/app/api/auth/login/route.ts:
- POST handler accepts { email, password }
- Validate with Zod schema
- Find user by email
- Verify password using verifyPassword()
- If valid:
  - Generate access token (15min) and refresh token (7 days)
  - Set cookies:
    - `token`: access token (httpOnly, secure, sameSite=strict, maxAge=15min)
    - `refreshToken`: refresh token (httpOnly, secure, sameSite=strict, maxAge=7days)
  - Return 200 with user session
- If invalid:
  - Return 401 with field-level error: `{ "success": false, "errors": { "password": "Email ou mot de passe incorrect" } }`

Security notes:
- Use httpOnly cookies (not localStorage) to prevent XSS
- secure flag for HTTPS in production
- sameSite=strict for CSRF protection

Test with curl and verify cookies are set.
  </action>
  <verify>
    <automated>curl -s -X POST http://localhost:3000/api/auth/login -H "Content-Type: application/json" -d '{"email":"test@example.com","password":"password123"}' -c - | grep -E "(token|refreshToken)"</automated>
  </verify>
  <done>Login endpoint validates credentials, sets httpOnly cookies with tokens, returns user session; returns 401 for invalid credentials</done>
</task>
