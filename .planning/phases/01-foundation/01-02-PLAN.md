## Technical Decisions
- Use `jose` instead of `jsonwebtoken` because Next.js App Router uses Edge runtime which doesn't support Node.js crypto module
- Use `bcryptjs` instead of `bcrypt` for better cross-platform compatibility
- Use Resend for email (free tier, good deliverability, simple API)

Install jose: `npm install jose`

Create src/types/auth.ts with interfaces:
```typescript
export interface UserSession {
  userId: string;
  email: string;
  name: string | null;
}

export interface AuthResponse {
  success: boolean;
  error?: string;
  session?: UserSession;
}
```

Create src/lib/jwt.ts with functions:
1. `signToken(payload: UserSession): Promise<string>` - Signs JWT with 15min expiry
2. `verifyToken(token: string): Promise<UserSession | null>` - Verifies and returns payload or null
3. `generateRefreshToken(userId: string): Promise<string>` - Long-lived token (7 days) for session persistence
4. `verifyRefreshToken(token: string): Promise<string | null>` - Returns userId or null

Use jose library (Edge-compatible):
- Algorithm: HS256
- Secret from process.env.JWT_SECRET
- Access token expiry: 15 minutes
- Refresh token expiry: 7 days

Create tests verifying:
- Token signs and verifies correctly
- Expired token returns null
- Tampered token returns null
